# kai-yun-pages

## 简介

`kai-yun-pages` 是一个用于归档与发布多个独立 HTML 页面的仓库。  
本仓库不针对任何特定域名或网站，仅作为静态页面的集中存放与版本管理空间。

通过 GitHub Pages 功能，可便捷地预览或分享仓库中存放的页面。

## 目录说明

```
kai-yun-pages/
├── index.html          # 仓库根目录页面
├── pages/              # 存放各独立 HTML 页面
│   ├── example-1.html
│   ├── example-2.html
│   └── ...
├── assets/             # 页面使用的资源文件（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
├── docs/               # 可选：文档或说明文件
└── README.md           # 本文件
```

- `pages/`：主要的页面存放目录，每个文件为一个独立页面。
- `assets/`：存放页面所需的样式、脚本、图片等资源，按类型分目录组织。
- `docs/`：可选的辅助文档或说明，非必需。

## 页面归档说明

- 所有页面均为独立的 HTML 文件，不依赖外部数据库或后端服务。
- 页面之间无强制关联，可单独访问。
- 建议在 `pages/` 目录下按主题或时间对页面进行子目录分类，便于管理。
- 每个页面应具备自包含的资源引用（相对路径），避免跨目录依赖。

## 使用方式

1. 克隆本仓库到本地。
2. 在 `pages/` 目录下添加或修改 HTML 文件。
3. 若页面需要额外资源，请放入 `assets/` 对应子目录。
4. 提交并推送至 GitHub 后，可通过仓库的 Pages 设置开启静态页面托管。
5. 开启后，页面可通过 `https://<用户名>.github.io/kai-yun-pages/pages/<文件名>` 访问。

> 更详细的使用说明请参考 [GitHub Pages 官方文档](https://docs.github.com/zh/pages)。
